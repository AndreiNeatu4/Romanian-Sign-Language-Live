================================================================================
LAST-MINUTE CHANGES - Moving-sign recognition overhaul
================================================================================

Goal: Improve recognition of moving-sign letters (J, Z, Q, S, Ș, Ț) and lay
the foundation for verb / full-word recognition without changing the user's
data-collection workflow (just add more videos to the existing folders).

Root cause of the previous poor moving-sign accuracy:
  Per-frame wrist re-centering erased the global trajectory, so the model
  only saw finger-pose changes, which for J vs I are nearly identical at
  every individual frame. The PATH the wrist/fingertip traces in space
  IS the sign for J, Z, Q, S, Ș, Ț - and that signal had been thrown away.


--------------------------------------------------------------------------------
NEW SHARED FEATURE MODULE (single source of truth)
--------------------------------------------------------------------------------

  data_preparation/feature_extractor.py

  Used by BOTH the offline pipelines and the live recognizer so the
  per-frame feature vector cannot drift between training and inference.


--------------------------------------------------------------------------------
PER-FRAME FEATURE LAYOUT (was 186, now 216)
--------------------------------------------------------------------------------

  [  0:126]  Hand local (wrist-centered, scale-normalized)
             2 hands x 21 landmarks x 3 coords        - same as before
  [126:186]  Face anchors (nose-centered, face-height-normalized)
             20 anchors x 3 coords                    - same as before
  [186:192]  Wrist GLOBAL position in body-anchor frame   *** NEW ***
             2 hands x 3 coords                       - the trajectory signal
  [192:198]  Index-fingertip GLOBAL position           *** NEW ***
             2 hands x 3 coords                       - the actual pen-tip
  [198:204]  Wrist velocity (delta wrist global)       *** NEW ***
             2 hands x 3 coords
  [204:210]  Wrist acceleration (delta velocity)       *** NEW ***
             2 hands x 3 coords
  [210:212]  Hand-present mask                          *** NEW ***
             1 = detected, 0.5 = bridged dropout, 0 = absent
  [212:216]  Reserved (zeros)                           - room for future feats

  Body-anchor frame: origin = shoulder midpoint, scale = shoulder-to-hip
  distance, sourced from MediaPipe Pose. Falls back to nose + face-height
  when pose is not detected.

  CRITICAL: the global channels are NOT re-centered per frame. They keep
  the trajectory across the whole sequence.


--------------------------------------------------------------------------------
FILE-BY-FILE CHANGES
--------------------------------------------------------------------------------

config.py
  - SEQUENCE_LENGTH    30  ->  45     (1.5 s @ 30 fps - enough for moving signs)
  - TARGET_FPS         (new) = 30     (resampling target, training & inference)
  - EXTRACT_POSE       False -> True  (needed for body anchor)
  - MODEL_TYPE         cnn_lstm -> cnn_bilstm_attn (new default)
  - USE_CLASS_WEIGHTS  (new) = True
  - GRAD_CLIP_NORM     (new) = 1.0
  - DIRECTIONAL_LETTERS (new) = {J, Z, Q, S, Ș, Ț}
       -> these letters are excluded from time-reverse augmentation
  - HAND_DROPOUT_BRIDGE_FRAMES (new) = 3
  - MIN_HAND_PRESENCE_RATIO    (new) = 0.5
  - FACE_MESH_ANCHORS pruned from 28 -> 20 entries (matches the trained model
    and the live recognizer's previous in-code anchor set)
  - TOTAL_FEATURES_SIZE constant added (= 216)

data_preparation/feature_extractor.py  [NEW FILE]
  - FrameFeatureExtractor: stateful per-stream extractor
      * MediaPipe handedness for stable left/right slot assignment
      * holds last-known landmarks for <= 3-frame dropouts
      * computes velocity / acceleration vs. previous frame
      * body-anchor (pose preferred, face fallback)
  - hand_presence_ratio(): quality metric for sequence filtering
  - resample_frame_indices(): source-fps -> target-fps resampling

data_preparation/extract_augmented_fast.py  [REWRITTEN]
  - Now uses FrameFeatureExtractor (single source of truth).
  - Resamples every video to TARGET_FPS so windows always = same wall-clock
    duration regardless of source FPS.
  - Drops sequences with < MIN_HAND_PRESENCE_RATIO hand-detected frames.
  - Skips time-reverse augmentation for DIRECTIONAL_LETTERS (J/Z/Q/S/Ș/Ț):
    a reversed J means something different.
  - Tighter speed range (0.9, 1.0, 1.1) since trajectory shape matters now.
  - More visual augmentations (10 pipelines: brightness/contrast, hue,
    CLAHE, gamma, Gaussian noise, motion blur, combos).
  - Albumentations API fix: passes images as a list (1.4+ requirement).

data_preparation/extract_gesture_data.py  [REWRITTEN]
  - Same FrameFeatureExtractor backbone as the augmented pipeline.
  - Same FPS resampling + quality filter.
  - Kept around as the lightweight no-augmentation alternative.

data_preparation/prepare_augmented_dataset.py  [REWRITTEN]
  - Splits by SOURCE VIDEO instead of by window.
      * The old per-window split leaked windows from the same video into
        both train and test - inflated accuracy artificially.
      * Per-video split = honest evaluation, especially with overlap=0.5.
  - Computes inverse-frequency class weights for the trainer.
  - Skips any stale .npy whose feature dim disagrees with the new layout.

training/train_model.py
  - New CNNBiLSTMAttnModel class (the new default):
      * LayerNorm input
      * 2x Conv1d (no maxpool - keeps full temporal resolution)
      * Bidirectional LSTM (was unidirectional)
      * Attention pooling over time (was: last-step only)
  - Class-weighted CrossEntropy via class_weights from dataset.pkl.
  - Gradient clipping (default norm 1.0) for attention stability.
  - On-the-fly landmark-space augmentation during training:
      rotation +/- ~6 deg, scale +/-5%, translation +/- 0.04, joint noise
      0.01, time-warp 0.85x..1.15x (30% chance per sample).
    Cheap (no MediaPipe re-extraction) and dramatically increases effective
    dataset size whenever the user records more videos.

application/realtime_recognition.py  [REWRITTEN]
  - Uses FrameFeatureExtractor - byte-identical features to training.
  - New CNNBiLSTMAttnModel mirrored from training (same state-dict keys).
  - Auto-detects feature dim and architecture from the checkpoint, so the
    existing 186-feature best_model.pth still loads (until retrain).
  - Subsamples MediaPipe + model inference to TARGET_FPS while still
    displaying the camera at 60 fps.
  - Motion detection uses the velocity channels (not mean-abs-diff over all
    features), so handshape jitter no longer masquerades as motion.
  - Draws shoulder / hip dots so the body-anchor lock is visible.


--------------------------------------------------------------------------------
HOW TO USE AFTER ADDING MORE VIDEOS
--------------------------------------------------------------------------------

  1. Drop new clips into  alphabet/<LETTER>/
        - 10-30 takes per moving letter is the rough target
        - vary signing speed, hand position, distance, signer location
        - DO NOT mirror/flip/time-reverse clips for J/Z/Q/S/Ș/Ț
          (the pipeline never does this either, but neither should you)

  2. Re-run the full pipeline:
        python run_pipeline.py

     This runs:
        a) data_preparation/extract_augmented_fast.py
              -> per-frame features at TARGET_FPS, sliced into 45-frame
                 windows with 50% overlap, multiplied by visual augmentations
                 and tight speed variations.
        b) data_preparation/prepare_augmented_dataset.py
              -> per-video stratified split, class weights computed
        c) training/train_model.py
              -> CNN+BiLSTM+attention with on-the-fly augmentation,
                 class-weighted loss, gradient clipping.

  3. Run the recognizer:
        python application/realtime_recognition.py


--------------------------------------------------------------------------------
SEQUENCE-LEVEL TRAINING (NOT FRAME-LEVEL)
--------------------------------------------------------------------------------

The pipeline trains on TEMPORAL SEQUENCES, not single images.

  Each video -> read all frames -> resample to TARGET_FPS -> extract one
  per-frame feature vector per frame -> slice into overlapping 45-frame
  windows -> each window is one training sample.

  The model (CNN -> BiLSTM -> attention) ingests the entire 45-frame window
  as a temporal signal. Every frame contributes; attention learns which
  frames matter most for each class.

  Quality filter drops any window with too few hand-detected frames so the
  model never trains on garbage.


--------------------------------------------------------------------------------
WHAT THIS UNLOCKS FOR VERBS / WORDS LATER
--------------------------------------------------------------------------------

  * Body-anchored trajectory + velocity/acceleration channels are exactly
    what verb-level signs need (e.g. signs that move toward / away from
    the body, two-handed signs that converge).
  * Sequence length 45 @ 30 fps = 1.5 s, which covers most lexical signs.
  * BiLSTM + attention generalizes to longer sequences without code change;
    the same architecture can be the encoder of a CTC / seq2seq head when
    you graduate to continuous sentence recognition.
  * The "hand-present mask" channel and the per-video split discipline
    are required infrastructure for sentence segmentation.
